Preciso corrigir inconsistências no fluxo de criação de evento de batizado, sincronização com financeiro e envio de mensagens via WhatsApp.

1. Botão de Finalização (Área do Aluno)

Na área do aluno:

Após definir a quantidade de parcelas do batizado, deve existir um botão “Finalizar”
Somente após clicar em Finalizar:
O evento deve ser oficialmente criado
A informação deve ser enviada ao painel administrativo
A ficha financeira deve ser gerada

Correção necessária:
Atualmente o fluxo não está garantindo corretamente essa finalização antes de enviar os dados.

2. Exclusão no Painel Administrativo (BUG CRÍTICO)

Ao excluir um evento de batizado no painel administrativo:

Deve acontecer:

Excluir o evento de batizado
Excluir automaticamente a ficha/notificação financeira vinculada a esse evento
Remover tudo também da área do aluno (sincronização total)

Problema atual:

O evento é excluído
Porém a notificação/ficha financeira continua existindo (erro grave de inconsistência)

Correção obrigatória:

Garantir exclusão em cascata (evento + financeiro + vínculo com aluno)
3. Envio de WhatsApp com Dados Corretos (Ficha Financeira)

Ao enviar a confirmação da ficha financeira via WhatsApp:

Deve incluir obrigatoriamente:

Modalidade escolhida pelo aluno (ex: número de parcelas)
Valor total
Valor das parcelas
Quantidade de parcelas
Datas (se aplicável)

Problema atual:

A mensagem está sendo enviada incompleta
Não está respeitando a modalidade escolhida pelo aluno

Correção obrigatória:

A mensagem deve ser dinâmica, baseada nos dados reais escolhidos pelo aluno
Não pode ser mensagem fixa/genérica
4. Regra Geral de Consistência
Tudo que for criado no aluno → refletir corretamente no admin
Tudo que for excluído no admin → apagar completamente no aluno
Nenhum dado financeiro pode ficar “órfão” no sistema
Objetivo da Correção:
Evitar inconsistência de dados financeiros
Garantir integridade do sistema
Melhorar a comunicação com o aluno
Garantir que o fluxo funcione do início ao fim sem falhas
