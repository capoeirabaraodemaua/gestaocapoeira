Considerando a ausência de backup completo dos dados antigos e os problemas atuais na plataforma, solicito a implementação de uma solução definitiva para reconstrução da base de dados, correção das inconsistências e normalização completa do sistema.

1. Criação de acesso pelo aluno (LOGIN SIMPLES)
Habilitar na área do aluno:
Cadastro via e-mail (obrigatório)
Criação de senha
Login com e-mail + senha
2. Fluxo dentro da área do aluno (OBRIGATÓRIO)
Após login, o sistema deve:
Identificar automaticamente o aluno
Puxar os dados já existentes (mesmo que incompletos)
Em seguida:
Exibir formulário de complementação de cadastro
O aluno preencherá os dados faltantes diretamente na área do aluno
3. Preenchimento e reconstrução dos dados
O aluno deve conseguir preencher:
Núcleo
Graduação
CPF
Endereço
Contato
Demais campos necessários
4. Sincronização automática (CRÍTICO)
Conforme o aluno for preenchendo os dados:
As informações devem ser salvas automaticamente no banco (Supabase)
Atualizar em tempo real o sistema
Esses dados devem:
Aparecer automaticamente no painel administrativo
Estar completos e atualizados
5. Controle de acesso por núcleo (OBRIGATÓRIO)
O sistema deve garantir:
Administrador geral → visualiza todos os alunos
Administrador de núcleo → visualiza apenas os alunos do seu núcleo
A separação por núcleo deve funcionar corretamente no sistema e no banco
6. Integração total entre sistema e banco
Garantir correspondência total entre:
Área de cadastro (frontend)
Tabela do banco (Supabase)
Caso haja divergência:
Criar ou ajustar colunas no banco
Padronizar estrutura
7. Estrutura do banco (ORIENTAÇÃO NECESSÁRIA)

Solicito que seja informado:

Quais colunas devem existir no Supabase
Estrutura ideal da tabela
Relacionamentos necessários
Ajustes técnicos necessários para evitar novos erros
8. Correções pendentes na plataforma

Ainda é necessário corrigir:

Separação dos alunos por núcleo
Exibição do núcleo
Exibição da graduação
Exibição do ID ACCBM
Botões “Ver” e “Editar” com dados completos
Frequência funcionando corretamente
9. Objetivo da solução
Reconstruir a base de dados sem depender de backup antigo
Garantir consistência total entre:
Área do aluno
Painel administrativo
Banco de dados (Supabase)
10. Importante
O processo deve ser simples para o aluno
Os dados devem ser sincronizados automaticamente
A plataforma deve voltar a funcionar de forma completa e estável
