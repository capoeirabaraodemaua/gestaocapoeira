Ajustar o sistema de cadastro de alunos para funcionamento multi-tenant com Supabase.

Objetivo:
Garantir que todo aluno seja salvo automaticamente vinculado ao tenant (núcleo), sem necessidade de preenchimento manual.

Regras obrigatórias:

1. Ao cadastrar um aluno, o sistema deve incluir automaticamente o campo:
   - tenant_id (UUID)

2. O tenant_id NÃO deve ser gerado automaticamente (não usar gen_random_uuid).
   Ele deve ser obtido com base no tenant do usuário logado ou do contexto atual da aplicação.

3. Implementar lógica para obter o tenant atual:
   - Se houver autenticação: usar o tenant_id associado ao usuário logado
   - Caso contrário: usar o tenant padrão definido no sistema

4. No momento do cadastro, enviar os dados no seguinte formato:

{
  "nome": "Nome do Aluno",
  "telefone": "21999999999",
  "data_nascimento": "2000-01-01",
  "tenant_id": "<ID_DO_TENANT_ATUAL>"
}

5. O campo tenant_id:
   - Deve ser obrigatório no backend
   - Não deve aparecer para o usuário na interface

6. Garantir validação:
   - Não permitir salvar aluno sem tenant_id
   - Evitar erro de foreign key com a tabela tenants

7. Ajustar listagem de alunos:
   - Filtrar automaticamente por tenant_id
   - Exibir apenas alunos do tenant atual

8. Garantir consistência:
   - Todos os novos registros devem ter tenant_id válido
   - Compatível com estrutura existente do Supabase (foreign key já configurada)

Resultado esperado:
Sistema multi-tenant funcional, onde cada núcleo acessa apenas seus próprios alunos, com cadastro automático e seguro.
